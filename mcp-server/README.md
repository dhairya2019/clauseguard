# ClauseGuard MCP Server

MCP server for legal contract analysis with India-specific awareness, powered by Claude.

## Features

- **Clause-by-clause risk assessment** -- HIGH / MEDIUM / LOW severity ratings
- **India-specific legal flags** -- S.27 non-compete, S.28 litigation restrictions, FEMA compliance, jurisdiction analysis
- **Missing clause detection** -- checks against expected clauses per document type
- **Safer alternative wording** -- suggests standard market language replacements
- **Structured JSON output** -- machine-readable results for integration into workflows
- **Multi-document support** -- freelance agreements, NDAs, ToS, employment contracts, SaaS agreements, vendor contracts

## Prerequisites

- [Node.js](https://nodejs.org/) 18 or later
- [Anthropic API key](https://console.anthropic.com/) with access to Claude

## Quick Start

```bash
# Set your API key
export ANTHROPIC_API_KEY=sk-ant-...

# Run via npx (no install needed)
npx -y clauseguard-mcp
```

The server communicates over stdio and is designed to be launched by an MCP client (Claude Desktop, Claude Code, or any MCP-compatible host).

## Claude Desktop Configuration

Add the following to your Claude Desktop config file:

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "clauseguard": {
      "command": "npx",
      "args": ["-y", "clauseguard-mcp"],
      "env": {
        "ANTHROPIC_API_KEY": "sk-ant-..."
      }
    }
  }
}
```

Replace `sk-ant-...` with your actual Anthropic API key.

## Claude Code Configuration

```bash
claude mcp add clauseguard -- npx -y clauseguard-mcp
```

Set `ANTHROPIC_API_KEY` in your shell profile (`~/.zshrc`, `~/.bashrc`, or equivalent) so it is available when Claude Code launches the server.

## Tool: `analyze_contract`

### Inputs

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `contract_text` | string | Yes | -- | Full text of the contract to analyze (min 50 characters) |
| `analysis_type` | enum | No | `"full"` | `"full"` = comprehensive, `"risk"` = risks only, `"summary"` = high-level overview |
| `party_perspective` | string | No | -- | Perspective for analysis: `"freelancer"`, `"client"`, `"employee"`, `"vendor"` |

### Output

Structured JSON with the following top-level fields:

| Field | Type | Description |
|---|---|---|
| `documentType` | string | Detected contract type (freelance_agreement, nda, terms_of_service, employment, saas, vendor, other) |
| `riskScore` | string | Overall risk: `"high"`, `"medium"`, or `"low"` |
| `riskSummary` | string | One-sentence summary of primary concerns |
| `riskBreakdown` | object | Counts of high, medium, and low risk clauses |
| `clauses` | array | Clause-by-clause analysis with risk, explanation, concerns, suggestion, and India-specific notes |
| `missingClauses` | array | Expected clauses not found in the document, with importance rating |
| `escalationFlags` | array | Clauses requiring immediate attention (FEMA, S.27, waiver of rights) |
| `disclaimer` | string | Legal disclaimer |

## Configuration

### Environment Variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `ANTHROPIC_API_KEY` | Yes | -- | Your Anthropic API key |
| `CLAUSEGUARD_MODEL` | No | `claude-sonnet-4-5-20250514` | Claude model to use for analysis |

## Development

```bash
# Clone and install
git clone <repo-url>
cd mcp-server
npm install

# Run in development mode (uses tsx, no build needed)
npm run dev

# Compile TypeScript to dist/
npm run build

# Open MCP Inspector for interactive testing
npm run inspect
```

### Project Structure

```
mcp-server/
  src/
    index.ts              # Entry point, tool registration, stdio transport
    analyze.ts            # Claude API integration with structured output
    errors.ts             # Domain error handling utilities
    schemas/
      contract.ts         # Input (Zod) and output (JSON Schema) definitions
    prompts/
      system-prompt.ts    # Analysis logic, risk rules, India-specific law
  dist/                   # Compiled output (generated by npm run build)
  package.json
  tsconfig.json
```
