  <action>
1. Install dependencies in the web-app directory:
   ```bash
   cd web-app && npm install @upstash/redis @fingerprintjs/fingerprintjs
   ```
   NOTE: Do NOT install @upstash/ratelimit — we use raw Redis INCR for lifetime counters (not time-window rate limiting). The research explicitly recommends against @upstash/ratelimit for this use case.

2. Create `web-app/src/lib/redis.ts` — Upstash Redis client and usage functions:
   - `redis` singleton via `Redis.fromEnv()` (reads UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN)
   - `getUserIdentifier(req: Request): string` — extracts IP from x-forwarded-for + fingerprint from X-Fingerprint header, returns `{ip}:{fingerprint}` compound key. Falls back to `{ip}:nofp` if no fingerprint header.
   - `checkAndIncrementUsage(identifier: string)` — INCR first (atomic), then check if > FREE_LIMIT (3). If over, DECR back. Returns `{ allowed, used, limit, remaining, isPaid }`. Checks paid status first via `paid:{identifier}` key.
   - `getUsageStatus(identifier: string)` — read-only version for status checks. Returns `{ used, limit, remaining, isPaid }`.
   - `markAsPaid(identifier: string)` — sets `paid:{identifier}` to true in Redis (no TTL — permanent).
   - Use constants: `FREE_LIMIT = 3`, `USAGE_PREFIX = "usage:"`, `PAID_PREFIX = "paid:"`.
   - Follow the exact code patterns from 04-RESEARCH.md Pattern 2.

3. Create `web-app/src/lib/fingerprint.ts` — Client-side fingerprint hook:
   - "use client" directive at top
   - `useFingerprint()` hook using dynamic import of @fingerprintjs/fingerprintjs
   - Cache visitor ID in module-level variable to avoid re-computation
   - Returns `string | null` (null while loading)
   - Follow 04-RESEARCH.md Pattern 5 exactly.

4. Create `web-app/src/lib/usage.ts` — localStorage helpers:
   - "use client" directive at top
   - `UsageData` interface: `{ used: number; limit: number; isPaid: boolean }`
   - `getLocalUsage()`, `setLocalUsage()`, `incrementLocalUsage()`, `markLocalPaid()`
   - Storage key: `"clauseguard_usage"`
   - Safe SSR handling (check `typeof window`)
   - Follow 04-RESEARCH.md Pattern 6 exactly.

5. Create `web-app/src/app/api/payment/status/route.ts` — Usage status endpoint:
   - GET handler that reads X-Fingerprint header and x-forwarded-for
   - Calls `getUserIdentifier(req)` then `getUsageStatus(identifier)`
   - Returns JSON with `{ used, limit, remaining, isPaid }`
   - This lets the client sync localStorage with server truth on page load (prevents Pitfall 5 from research).
