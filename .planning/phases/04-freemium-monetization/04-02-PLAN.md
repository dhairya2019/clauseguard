2. Create `web-app/src/types/razorpay.d.ts` — Type declarations for the Razorpay checkout.js client:
   - `RazorpayResponse` interface: `razorpay_payment_id`, `razorpay_order_id`, `razorpay_signature` (all strings)
   - `RazorpayOptions` interface: key, amount, currency, name, description, order_id, handler, prefill?, theme?, modal?
   - `RazorpayInstance` interface: open(), close(), on()
   - `declare global { interface Window { Razorpay: new (options: RazorpayOptions) => RazorpayInstance } }`
   - Follow the exact types from 04-RESEARCH.md Code Examples.

3. Create `web-app/src/app/api/payment/create-order/route.ts`:
   - Import `Razorpay` from "razorpay", import `getUserIdentifier` from `@/lib/redis`
   - Instantiate Razorpay with `process.env.RAZORPAY_KEY_ID` and `process.env.RAZORPAY_KEY_SECRET`
   - POST handler: extract `identifier` from request body
   - Create order with `razorpay.orders.create({ amount: 29900, currency: "INR", receipt: "cg_{timestamp}", notes: { identifier } })`
   - CRITICAL: Store identifier in order notes for Pitfall 4 prevention (replay attack protection)
   - Return JSON: `{ orderId: order.id, amount: order.amount, currency: order.currency, keyId: process.env.NEXT_PUBLIC_RAZORPAY_KEY_ID }`
   - Wrap in try/catch, return 500 on failure.

4. Create `web-app/src/app/api/payment/verify/route.ts`:
   - Import `crypto` from "crypto", import `markAsPaid` from `@/lib/redis`
   - POST handler: extract `razorpay_order_id`, `razorpay_payment_id`, `razorpay_signature`, `identifier` from body
   - Validate all fields present, return 400 if missing
   - Compute expected signature: `crypto.createHmac("sha256", RAZORPAY_KEY_SECRET).update("{order_id}|{payment_id}").digest("hex")`
   - Use `crypto.timingSafeEqual` for comparing signature (prevent timing attacks). Convert both strings to Buffer first.
   - If signatures don't match, return 400 `{ error: "Invalid signature" }`
   - If valid, call `markAsPaid(identifier)` to set paid flag in Redis
   - Return 200 `{ success: true, message: "Payment verified, quota unlocked" }`

5. Create `web-app/src/components/freemium/upgrade-modal.tsx`:
   - "use client" directive
   - Import `Script` from "next/script" for loading checkout.js
   - Import shadcn Dialog, DialogContent, DialogHeader, DialogTitle, DialogDescription from `@/components/ui/dialog`
   - Import Button from `@/components/ui/button`
   - NOTE: The Dialog component may not be installed yet. If it is not in `web-app/src/components/ui/`, install it via `npx shadcn@latest add dialog` before creating this file.
   - Props: `{ open, onOpenChange, identifier, onPaymentSuccess }`
   - `handlePayment` async function:
     a. POST to /api/payment/create-order with { identifier }
     b. Extract orderId, amount, currency, keyId from response
     c. Build RazorpayOptions with handler callback that POSTs to /api/payment/verify
     d. On successful verification, call onPaymentSuccess() and close modal
     e. Check `window.Razorpay` exists before calling `new window.Razorpay(options).open()`
     f. Show error state if Razorpay script not loaded
   - Render: Script tag for checkout.js (strategy="lazyOnload"), Dialog with pricing card (INR 299, one-time, features list), "Pay with Razorpay" button
   - Theme color: "#1e40af" to match ClauseGuard blue
   - Follow the exact component from 04-RESEARCH.md Code Examples.
