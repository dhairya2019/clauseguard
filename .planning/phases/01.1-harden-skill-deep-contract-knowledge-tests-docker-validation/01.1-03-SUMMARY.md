## Decisions Made
- Used realistic contract language (84-128 lines each) rather than minimal test stubs, ensuring tests validate against production-like complexity
- Inline contract content passed to claude via stdin-style prompt rather than file path reference, improving Docker portability and avoiding path resolution issues
- Machine-readable counter output (PASS_COUNT/FAIL_COUNT/WARN_COUNT) from validate.sh enables future CI pipeline integration
- MUSTNOT patterns (e.g., "GREEN document" for RED-profile contracts) prevent false-positive classification
