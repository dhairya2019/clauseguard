Add a new "Section 4: Edge Case Detection Patterns" to clause-patterns.md AFTER the existing Section 3 (Safer Alternative Templates). This section teaches the skill to detect hidden, disguised, and compound risks.

Include these 6 edge case categories with specific detection guidance:

**1. Multi-Clause Interaction Risks** (table format)
| Combination | Combined Risk | Detection Keywords |
- Non-compete + IP assignment = cannot work in field AND client owns tools
- Indemnification + no liability cap = unlimited financial exposure
- Termination for convenience + no in-progress payment + IP assignment on termination = client gets everything, pays nothing
- Auto-renewal + price escalation + long cancellation notice = locked into increasing costs
- Broad confidentiality + broad non-solicitation = disguised non-compete

**2. Buried Clauses in Boilerplate** (table format)
| Location | What Gets Buried | Detection Guidance |
- Exclusivity buried in "Definitions" section
- Non-compete buried in "General Provisions" or "Miscellaneous"
- IP assignment buried in "Confidentiality" section
- Indemnification hidden in "Representations and Warranties"
- Auto-renewal buried in "Term" section without clear heading

Detection guidance: "When analyzing contract, do NOT skip sections titled 'General', 'Miscellaneous', 'Definitions', or 'Boilerplate'. These sections frequently contain substantive obligations disguised as administrative clauses."

**3. Misleading Headings** (table format)
| Heading | What It Actually Contains | Red Flag |
- "Intellectual Property Protection" -> assigns all IP away from creator
- "Mutual Obligations" -> entirely one-sided terms
- "Standard Terms" -> non-standard, aggressive clauses
- "Flexible Engagement" -> locks in exclusivity
- "Performance Standards" -> contains termination triggers

Detection guidance: "Always read clause content regardless of heading. If heading suggests mutual/standard/protective but content is one-sided/aggressive, flag the mismatch as a risk indicator."

**4. Compound Sentences Hiding Risk** (table format)
| Example Pattern | Hidden Risk | Detection Keywords |
- "assigns all IP AND agrees not to compete" = two risks in one sentence
- "upon completion AND acceptance at sole discretion" = payment trap in conjunction
- "includes all materials, tools, ideas, concepts, and methodologies" = scope creep through list expansion
- "may terminate for cause, including but not limited to" = open-ended termination triggers

Detection guidance: "Break compound sentences at AND/OR/including/as well as conjunctions. Analyze each clause fragment independently for risk."

**5. Defined Term Manipulation** (table format)
| Defined Term | Overbroad Definition | Risk |
- "Confidential Information" defined to include publicly available data
- "Deliverables" defined to include future work not yet scoped
- "Compensation" defined to exclude certain types of work
- "Termination for Cause" defined so broadly any minor issue qualifies
- "Competing Business" defined to cover entire industry

Detection guidance: "Check the Definitions section for terms used in restrictive clauses. An unreasonable definition can make an otherwise reasonable clause dangerous."

**6. Cross-Reference Traps** (table format)
| Pattern | Risk | Detection Keywords |
- "Subject to Section 12" where Section 12 has aggressive carve-outs
- "As defined in Exhibit B" where Exhibit B is not provided
- "In accordance with Company Policy" where policy can change unilaterally
- "Per the rates in Schedule A" where Schedule A is blank or TBD

Detection guidance: "Verify all cross-references resolve to actual content. Flag any reference to missing exhibits, undefined policies, or TBD schedules as YELLOW risk."

Keep the same formatting style (tables with clear columns) used in the rest of the file. End section with the existing footer note.
